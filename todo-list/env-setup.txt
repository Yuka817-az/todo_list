# 環境変数設定手順

## 1. .envファイルの作成

プロジェクトルート（todo-listディレクトリ）に .env ファイルを作成してください。

例：
- パス: C:\Users\peri-\.cursor\todo_list\todo-list\.env

## 2. データベースURLの設定

### MySQL を使用する場合
DATABASE_URL="mysql://root:password@localhost:3306/todo_db"

### PostgreSQL を使用する場合
DATABASE_URL="postgresql://postgres:password@localhost:5432/todo_db"

## 3. 実際の値に置き換える

上記の例では以下の値が含まれています：
- root: データベースのユーザー名
- password: データベースのパスワード
- localhost: データベースのホスト
- 3306 (MySQL) または 5432 (PostgreSQL): ポート番号
- todo_db: データベース名

これらの値を実際の環境に合わせて変更してください。

## 4. ターミナルでの一時設定（開発用）

ターミナルで環境変数を設定することもできます：

### PowerShell（Windows）の場合
$env:DATABASE_URL="mysql://root:password@localhost:3306/todo_db"

### Bash/Linux の場合
export DATABASE_URL="mysql://root:password@localhost:3306/todo_db"

ただし、この方法では端末を閉じると設定が失われます。

## おすすめ

開発環境では .env ファイルを使用することをお勧めします。
- 設定が永続化される
- バージョン管理から除外される（.gitignoreに含まれている）
- チームメンバーと共有しやすい
